## 1. Re-read of the code — actual resolution (confirmed)
Gate = `ThreadService::isDealLimitReached` = **personal-limit OR business-limit** (`:1715`).
- **Personal/user path** (`:1742`) applies only if the user has a `DealLimits` override — **user-level limits are "Disabled functionality"**, so normally inert.
- **Business path** (`BusinessService::checkIfLimitReached`, `:646`):
  - base = **business `DealLimits` row** for the country if present, **else country default** (`getBusinessDealLimits`, `:758-763`);
  - if an **active package** has a limit → effective = **`max(packageLimit, base)`** (`:661-665`);
  - blocked if `posted >= effective` (`:668`).

So the **intended commercial model** (package > business-level > country default; `0` default = paywall for unpaid/unconfigured businesses) is **functionally what the code does** — with the nuance that it's a `max()`-combine over a *business-or-country-default* base, not a strict short-circuit (with a `0` country default, the package/allowance is what raises the cap). Documented both the intent and the exact behaviour, with the reconciliation note.
