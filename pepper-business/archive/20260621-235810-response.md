## Deprecations found in our code — all fixed

### 1. Implicitly-nullable parameter types — PHP **8.4** deprecation (6 sites)
`Type $x = null` is deprecated; fix is the explicit `?Type $x = null`. Behavior is identical (the param was already nullable via its `null` default; docblocks already said `|null`). Pure signature change.

### 2. `Reflection*::setAccessible()` — PHP **8.5** deprecation (5 test files, 11 calls)
Since PHP 8.1 reflection ignores accessibility, so `setAccessible(true)` is a **no-op** and is deprecated in 8.5. The smallest safe fix is to delete the calls — the surrounding `->invoke()` / `->getValue()` works unchanged.

`tests/unit/AdManager/AliExpressTargetingTest.php` (×6), `tests/unit/Invoice/InvoiceDataOcularEventTest.php`, `tests/unit/Clicky/ValidateClickyProgramLogJobTest.php` (×2), `tests/unit/Clicky/ProgramTrackingLinkRuleTest.php`, `tests/helpers/Unit/OcularEventAware.php`.

**No other deprecated constructs found** — the many `->each(` hits were Laravel Collection methods (false positives), not the removed PHP `each()`.
