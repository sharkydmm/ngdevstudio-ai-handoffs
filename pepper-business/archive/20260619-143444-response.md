## The one question (next highest-value ambiguity)

**What I found.** Spain (`ES`) is hardcoded out of **both** invoice-generation paths, yet it is an open deal-submission market.

**Where.**
- Standard invoicing: `app/Services/Invoice/GenerateInvoiceBaseService.php:608-610` — `shouldSkipInvoiceGeneration($country) { return in_array($country, ['ES']); }`.
- Loyalty invoicing: `app/Services/Invoice/GenerateLoyaltyProgramInvoiceService.php:86` — only generates for `{DE, NL, GB, FR, PL}`, so `ES` is skipped there too.
- Yet `config/countries.php` (ES block) has **open deal submission** (`max_deals_per_week/day = 5`) **and** configured invoice thresholds (`invoice_threshold = 100`, `loyalty_program_invoice_threshold = 15`).

**What I think it does.** Spain accepts deal submissions (and presumably serves ad inventory), but **no invoices are ever generated for ES** — both the standard and loyalty generation services deliberately skip it.

**Why it looks unusual.** A market that is *open for submissions* and has *invoice thresholds configured* but produces *zero invoices* looks like lost billing or an incomplete launch. The skip is **hardcoded in code** (a literal `['ES']`), so it's invisible from `config/countries.php` — which actually suggests ES *should* invoice (it has thresholds). The two signals contradict each other.
